# zztop — Requirements

**Date:** 2026-06-03
**Status:** Brainstorm complete, ready for planning
**Scope tier:** Standard

## Problem

`zsh-z`'s `z <pattern>` flow works well when the user remembers a fragment of the target path. It's less useful when:

- The user wants to *browse* recent/frequent directories without recalling a pattern.
- Picking visually is faster than typing a pattern + Enter.

The user loves `z` and does not want to replace it. They want a companion command that surfaces the top frecency-ranked paths and lets them pick one with arrow keys.

## Users

- **Primary:** the author — a heavy `zsh-z` user on macOS who lives in the terminal across a handful of working directories per day.
- **Secondary:** other zsh + zsh-z users who'd benefit from a "show me where I've been" picker. Not optimized for, but the design shouldn't preclude.

## Goals

- Augment `zsh-z` with a list-and-pick command. One keystroke to invoke (`zz`), arrow keys to choose, Enter to jump.
- Stay lightweight: small Rust binary, no fuzzy filter, no preview pane, no config.
- Reuse `zsh-z`'s existing frecency data — no duplicate tracking, no separate database.

## Non-goals

- Replacing or modifying `z`. The binary never writes to `zsh-z`'s datafile.
- Cross-shell support (bash, fish). Zsh-only for v1.
- Fuzzy filtering or type-to-search. Arrow keys + Enter only.
- Configurability (list size, ranking algorithm, alternate datafiles, themes). Hardcoded defaults, change in source if needed.
- Tracking or storing anything itself.

## Behavior

### Invocation

User runs `zz` in a zsh shell. `zz` is a small zsh function (shipped as a snippet for the user's `.zshrc`) that:

1. Calls the `zztop` binary, capturing stdout.
2. If the binary exits 0 with a non-empty path, `cd`s to it.
3. If the binary exits non-zero (Esc / no selection), does nothing.

### List

The binary reads `zsh-z`'s datafile, ranks entries by `zsh-z`'s frecency formula (recency × frequency), and renders the top 5 in a TUI:

```
❯ 1  ~/Workspace/experiments/zztop
  2  ~/Workspace/experiments/worktree-cli
  3  ~/Workspace/personal/aeraline
  4  ~/.claude/plugins/cache
  5  ~/Workspace/dotfiles

  [↑↓ move, ⏎ select, esc cancel]
```

- Paths are shown with `~` substituted for `$HOME` for readability.
- Cursor (`❯`) starts on row 1 (top-ranked).
- ↑/↓ moves the cursor; wrapping at top/bottom is acceptable but not required.
- Enter prints the selected absolute path to stdout and exits 0.
- Esc / Ctrl-C exits non-zero with no stdout output.

### Datafile

- Read path resolution: `$_Z_DATA` if set, otherwise `~/.z`.
- Format: `zsh-z`'s `path|rank|time` lines, one per directory.
- If the file is missing or empty: print a friendly message to stderr ("no zsh-z data found") and exit non-zero.
- If a top-ranked path no longer exists on disk: skip it silently and continue down the list. (We don't want to jump into a stale path.)

## Success criteria

- Running `zz` shows 5 paths in under ~50ms on a warm cache.
- Arrow keys + Enter select the right path; Esc cancels cleanly.
- Selecting a path lands the user in that directory in their current shell.
- Works on macOS with a recent zsh; binary is a single static-ish executable.

## Dependencies & assumptions

- **`zsh-z` datafile format is stable.** `zsh-z` writes `path|rank|time` per line; if upstream changes this, we update the parser. (Low risk — the format hasn't changed in years.)
- **Top 5 is the right number.** If it isn't enough, bumping a constant is a one-line change. We're not adding configuration to defer the decision.
- **The shell wrapper pattern is the right `cd` mechanism.** A child process can't change the parent shell's directory; capturing stdout in a function is the standard solution.

## Out of scope (deferred)

- Bash / fish integration. Same wrapper pattern would work but isn't shipped.
- Fuzzy filter mode (`--filter`, `zz <pattern>`). If picking from 5 stops being enough, this is the natural next step.
- Bumping list size dynamically based on terminal height.
- A `--print` mode that just lists paths without the TUI.

## Open questions for planning

- Which TUI crate? (`crossterm` + hand-rolled rendering vs. `ratatui` vs. `dialoguer`/`inquire`. The UI is small enough that `crossterm` alone is plausible.)
- Binary name and install path. Crate name `zztop`; binary `zztop`; the zsh wrapper function is `zz`.
- Repository layout (single crate vs. workspace). Single crate likely fine.
- Where to ship the `zz()` snippet — README only, or a `zztop init zsh` subcommand that prints it for `eval "$(zztop init zsh)"`.
